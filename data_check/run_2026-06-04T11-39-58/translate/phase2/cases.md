# 测试用例：用户登录并进入工作台

## 1. 业务背景与初始状态

本测试用例验证用户通过手机号和密码登录平台的完整流程。登录前，用户处于未认证的登录页面，需输入有效的手机号、密码，并同意用户协议后提交登录请求。登录成功后，系统将用户重定向至工作台首页，展示个性化菜单和用户信息。

## 2. 测试步骤流

### [步骤 1] 输入用户名手机号
- **执行动作**：在「用户名」输入框中输入手机号 15700078644
- **状态验证**：输入框值从空变为 15700078644，表单状态已更新

### [步骤 2] 输入登录密码
- **执行动作**：在「密码」输入框中输入密码
- **状态验证**：输入框值从空变为掩码显示，表单状态已更新

### [步骤 3] 勾选用户协议复选框
- **执行动作**：点击「用户协议」复选框
- **状态验证**：复选框状态从未选中变为已选中，表单状态已更新

### [步骤 4] 勾选同意协议复选框
- **执行动作**：点击「同意协议」复选框
- **状态验证**：复选框从未选中变为已选中，表单状态已更新

### [步骤 5] 提交登录表单
- **执行动作**：点击「立即登录」按钮
- **状态验证**：登录页跳转至工作台页面，左侧显示「我的对话」「Agent」「我的应用」等菜单，顶部显示用户名「余泽超」及「黄金会员」标签，主体区域展示多个应用卡片，底部出现「积分签到」「签到成功」提示

# 测试用例：专家Agent选择与智能问数对话交互

## 1. 业务背景与初始状态
用户已完成系统偏好设置配置，现需切换至专家模式，选择智能问数专家进行咨询对话，并完成首次问答交互。

## 2. 测试步骤流

### [步骤 20] 进入专家模式选择界面
- **执行动作**：点击「专家模式」选项
- **状态验证**：页面左侧显示多个专家Agent选项卡片，包括：AI操作优化专家、AI复杂控制专家、AI回路整定专家、AI生产参数预测专家、AI生产异常检测专家、预测控制专家、回路整定专家、能耗瓶颈分析专家、AI换热网络优化专家、工业知识专家、报表分析专家、智能问数专家

### [步骤 21] 选择智能问数专家Agent
- **执行动作**：点击「智能问数与结果分析」Agent选项
- **状态验证**：页面切换至智能问数专家模式，左侧专家选项卡片收起，输入框变为可输入状态，显示标题「智能问数专家」

### [步骤 22] 输入咨询问题
- **执行动作**：在「输入内容进行提问」输入框中输入文本「有哪些可以问」
- **状态验证**：输入框内容更新为「有哪些可以问」，表单状态已更新

### [步骤 23] 发送问题并获取AI回复
- **执行动作**：点击发送按钮（SendOutlined图标）
- **状态验证**：页面创建新对话项「有哪些可以问」，AI助手返回智能问数专家可咨询的三类问题说明：能耗与物耗指标查询、设备位号数据查询、系统与规则咨询

### [步骤 24] 对AI回复进行满意度反馈
- **执行动作**：点击AI回复下方的点赞按钮
- **状态验证**：点赞图标变为打勾确认状态，显示提示文本「感谢您的赞美与肯定」

### [步骤 25] 切换至Agent管理视图
- **执行动作**：点击左侧导航栏中的会话记录项或Agent管理入口
- **状态验证**：页面从对话聊天视图切换至Agent管理视图，出现分段控制器（对话生成/组态生成）、搜索Agent输入框、导入Agent按钮及离线/实时模式切换功能

# 测试用例：创建AI控制类Agent

## 1. 业务背景与初始状态
用户已登录系统，当前位于组态生成页面，需创建一个新的AI控制类Agent。该Agent关联特定装置，并将在创建成功后出现在Agent列表中。

## 2. 测试步骤流

### [步骤 26] 切换至组态生成模式
- **执行动作**：点击「组态生成」分段控件标签
- **状态验证**：页面保持当前状态

### [步骤 27] 查看组态生成模式下的Agent列表
- **执行动作**：点击「组态生成」单选按钮
- **状态验证**：分段控制器选中状态切换至组态生成；页面显示Agent卡片列表，包含「仪表-离线」等预设Agent项，并出现新增Agent入口及启用配置操作按钮

### [步骤 28] 筛选MPC类型Agent
- **执行动作**：点击分类筛选区域中可见文本为「控制」的标签
- **状态验证**：列表视图切换为显示「全部/MPC/AI」标签，并出现「uu私有控制MPC」和「ai1私有控制MPC」两个Agent卡片项

### [步骤 29] 筛选AI控制类型Agent
- **执行动作**：点击可见文本为「AI」的分类标签
- **状态验证**：列表视图筛选为AI控制类Agent，显示「AI_Control_1」卡片，状态显示「模型配置失败」「推理验证」「启用配置未完成」

### [步骤 30] 打开创建Agent对话框
- **执行动作**：点击可见文本为「新增Agent」的按钮
- **状态验证**：页面弹出「创建Agent」模态对话框，包含Agent名称、Agent描述、Agent类型（控制）、对应装置、是否有模型（默认否）、数据来源（默认数据文件上传）、是否需要引导上传数据（默认否）、保存目录（TPT智能体）等表单项，以及取消和确认按钮

### [步骤 31] 填写Agent名称
- **执行动作**：在Agent名称输入框中输入"ai_demo"
- **状态验证**：输入框已填入值"ai_demo"，表单状态更新

### [步骤 32] 选择对应装置
- **执行动作**：在对应装置下拉框中选择"ai_equip"
- **状态验证**：对应装置字段已选为"ai_equip"，表单状态更新

### [步骤 33] 提交创建Agent表单
- **执行动作**：点击"确认"按钮
- **状态验证**：创建Agent模态对话框关闭，页面跳转至iframe内容区域
