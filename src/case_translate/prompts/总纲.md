# LLM Prompt 总纲

> 说明：本系统**只有「翻译」链路会调用 LLM**。录制、预处理均不调用。  
> **Skill Prompt** 在 `prompts/md/*-skill.md`（LangGPT 9 维，共 3 份精华）；**User** 为代码内动态拼接，见各 `*.js`。  
> LLM 全量审计落盘：`run_*/llm_audit/`。

---

## 一、整体调用顺序

```
录制完成 (meta + actions + snapshots)
        ↓
    预处理（无 LLM）
        ↓
┌─ snapshots → steps ──────────────────────────────────┐
│  Skill: snapshots-2-steps-skill.md（LangGPT 9维）      │
│  User: step-structured.js 内嵌拼接                    │
│  → step_2_structured_steps.json                    │
└────────────────────────────────────────────────────┘
        ↓
┌─ steps → cases ────────────────────────────────────┐
│  Skill: steps-2-cases-skill.md                       │
│  User: case-generation.js 内嵌拼接                   │
│  → AI_cases.md                                     │
└────────────────────────────────────────────────────┘
        ↓
┌─ case → agents ────────────────────────────────────┐
│  Skill: case-4-agents-skill.md                       │
│  User: agent-txt.js 内嵌拼接                         │
│  → case_4_agents.txt                               │
└────────────────────────────────────────────────────┘
```

**触发入口：** Dashboard「AI 翻译」、`npm run translate`、独立翻译 EXE。

（代码审计字段仍为 `phase1` / `phase2` / `phase4`，与产物文件名 `step_2` / `AI_cases` / `case_4` 对应。）

---

## 二、snapshots → steps（`snapshots-2-steps-skill.md`）

### 什么时候用

- 预处理完成后，对**每一条有效 action**（非 skip、非 noise）调用 LLM。
- 默认 **每批 3 条** action 合并为一次请求（微批处理）。
- 每批发送 **1 次** skill + user，期望返回 **N 条**结构化步骤（N = 本批 action 数）。

### 使用的文件

| 来源 | 角色 | 何时加载 |
|------|------|----------|
| `md/snapshots-2-steps-skill.md` | Skill | **每次本段主调用必用** |
| `step-structured.js` → `buildUserPrompt` | User（历史上下文 + action 证据 JSON） | **每次本段主调用必用** |

### User 拼接内容（改代码，非 md）

- `contextHistory`：本批之前最近 3 条已解析步骤摘要
- `actionBlocks`：本批每条 action 的 element / snapshotDiff / formStateDelta 等

### LLM 要做什么

把物理操作 + 证据翻译成结构化字段：`description`、`uiChange`、`page`、`actionKind`、`target`、`basis`、`confidence` 等，写入 `parsedSteps[]`。

### 产物

`run_*/step_2_structured_steps.json`

### 失败策略

整批 JSON 无法解析时：记入 `llm_audit` 与 `step_2_structured_steps.errors.json`，对应 index 走 `buildFallbackStructuredStep`（非 LLM repair）。

## 三、steps → cases（`steps-2-cases-skill.md`）

### 什么时候用

- 结构化步骤完成后，对 **有效步骤**（`status=normal` 或 `fallback`）做 Case 归纳。
- **滑动窗口**调用：每轮取最多 20 条步骤，LLM 输出 **1 个 Case**，然后按 `consumeStepCount` 向前滑动，直到全部消费完。
- 一轮录制通常触发 **多轮** LLM 调用（例如 4 轮 → 4 个 Case）。

### 使用的文件

| 来源 | 角色 | 何时加载 |
|------|------|----------|
| `md/steps-2-cases-skill.md` | Skill | **每轮必用** |
| `case-generation.js` → `buildPhase2WindowUserPrompt` | User（窗口步骤 JSON + index 列表） | **每轮必用** |

### LLM 要做什么

从窗口步骤中归纳 **1 个**测试用例（title、summary、步骤表），且只覆盖窗口 **前缀连续** 的一部分步骤。

### 产物

`run_*/AI_cases.md`

### 调用参数

`temperature=0.3`，无 json_schema；本地 `parseSingleCaseJsonResponse` 解析 JSON。

## 四、case → agents（`case-4-agents-skill.md`）

### 什么时候用

- 结构化步骤完成后（与 cases 段并行依赖 `step_2`，不依赖 `AI_cases` 结果）。
- 对有效步骤做 **业务逻辑聚合**，生成供下游自动化 Agent 执行的文本用例。
- **滑动窗口**：默认每窗最多 20 条；通常 **1 次**调用即可覆盖整段录制（步骤少时）。

### 使用的文件

| 来源 | 角色 | 何时加载 |
|------|------|----------|
| `md/case-4-agents-skill.md` | Skill | **每次本段主调用必用** |
| `agent-txt.js` → `buildAgentTxtUserPrompt` | User（本窗步骤 JSON） | **每次本段主调用必用** |

### LLM 要做什么

输出 JSON：`useCaseName`、`useCasePurpose`、`agentSteps[]`（含 logicalName、microActions、consumeStepCount）。  
本地代码再把 JSON **渲染**成 `case_4_agents.txt` 纯文本。

### 产物

`run_*/case_4_agents.txt`

### 调用参数

`temperature=0.1`，无 json_schema。JSON 解析失败时走本地 1:1 兜底（`agent-txt-generator.js`），不再调用 repair LLM。

## 五、快速对照：改哪份 Skill 解决什么问题

| 你想优化… | 改哪里 |
|-----------|--------|
| 单步语义描述不准（输入/点击/UI 变化） | `snapshots-2-steps-skill.md` |
| LLM 合并步骤、丢 index | `snapshots-2-steps-skill.md`（N 进 N 出规则） |
| basis / 证据引用质量 | `snapshots-2-steps-skill.md` Output Format |
| 整批返回不是合法 JSON | 查 `llm_audit`；调主 Skill 或解析逻辑（无 repair） |
| 传给模型的 action 证据格式 | `step-structured.js` → `buildUserPrompt` |
| Case 标题、摘要、步骤表格质量 | `steps-2-cases-skill.md` |
| Case 切分太早/太晚 | `steps-2-cases-skill.md` |
| Agent 用例逻辑步骤划分 | `case-4-agents-skill.md` |
| Agent 微观动作缺少定位信息 | `case-4-agents-skill.md` |
| Agent 段输出非 JSON | `case-4-agents-skill.md`；失败见 audit / 本地兜底 |

## 六、不涉及 LLM 的阶段（勿找 Prompt）

| 阶段 | 说明 |

| 预处理 | 归并、diff、enriched |
